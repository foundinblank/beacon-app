# Ping (Auto-Center + Ripple) Implementation Plan

**Goal:** Add a "Ping" feature that auto-centers the cursor and/or plays a contracting ripple animation, triggered by Cmd-Shift-/ global hotkey or menu bar.

**Architecture:** GlobalHotkeyManager registers a Carbon hotkey, AppDelegate.performPing() orchestrates the action, RippleAnimationManager renders 3 contracting CAShapeLayer rings in OverlayView. Follows existing renderer pattern (CAShapeLayer + CABasicAnimation + UserDefaults observer).

**Tech Stack:** Swift 6, AppKit, Core Animation (CAShapeLayer + CABasicAnimation), Carbon (RegisterEventHotKey), os.Logger

**Spec:** `docs/superpowers/specs/2026-03-14-ping-design.md`

**Important — Xcode project:** This project uses explicit source file references in `Beacon.xcodeproj/project.pbxproj`. Every new `.swift` file must be added to the Xcode project.

## Bugs Fixed During Implementation

1. **SwiftUI delegate wrapping:** `NSApp.delegate` is SwiftUI's internal delegate, not the app's `AppDelegate`. Fixed by passing real `AppDelegate` from `@NSApplicationDelegateAdaptor` directly to `MenuBarMenuContent`.
2. **Caps Lock double-tap only counted off→on transitions:** Each press toggles state, so two taps only produce one on-transition. Fixed by counting any state change as a tap.
3. **Synthetic resetCapsLock events poisoned timing:** `resetCapsLock()` generated flagsChanged events that set `lastTapTime`, causing every-other-attempt failures. Fixed with a `resetting` guard flag.
4. **Ultimately replaced CapsLockDetector with GlobalHotkeyManager** using Cmd-Shift-/ for reliable triggering.
